# 番茄钟桌面应用 PRD

**版本**：v0.1（与 BRD `brd-4-28.md` 对齐）  
**状态**：草案，待评审  
**范围**：MVP 以 macOS 为主，架构预留 Windows 分发能力  

---

## 1. 文档目的与范围

本 PRD 在《需求目标》（BRD）基础上，将目标拆解为可验收的功能、数据模型、交互与技术约束，便于排期、实现与测试。不包含具体代码仓库结构或 CI 配置细节（可在技术方案中展开）。

---

## 2. 产品概述

### 2.1 问题陈述

市面番茄钟产品常存在：收费门槛、功能与个人工作流不匹配、数据无法与自有任务体系统一等问题。

### 2.2 产品定位

一款**小巧、常驻桌面**的个人番茄钟客户端：计时能力与飞书多维表格（Bitable）打通，使「专注记录」与「任务状态」同源，便于回顾与统计。

### 2.3 目标用户

单人自用（主）；未来可扩展为多账号，但 **MVP 不承诺多租户**。

### 2.4 成功标准（MVP）

| 维度 | 标准 |
|------|------|
| 计时 | 可完成一次标准番茄（默认 25 分钟），支持开始/暂停/放弃；系统休眠策略在 PRD 第 7 节约定 |
| 结束闭环 | 每个番茄正常结束时，可写备注、可选关联任务、可更新任务状态；数据写入飞书对应表 |
| 离线 | 网络不可用时本地队列缓存，恢复后自动补写（见 6.3） |
| 分发 | macOS 提供可双击安装或拖拽安装包；用户无需单独安装 JDK |

## 3. 用户场景与核心流程

### 3.1 场景 A：开始专注

1. 用户打开应用（或从菜单栏/托盘唤起 compact 窗口）。
2. 选择「专注时长」（默认 25m；可选预设 15/25/45 或自定义整数分钟，**MVP 可仅 25m**）。
3. 可选：从任务列表中「钉选」本番茄要关联的任务（也可留到结束再选）。
4. 点击开始；界面显示剩余时间与轻量进度。

### 3.2 场景 B：番茄正常结束

1. 计时器归零；播放简短提示音/系统通知（可关）。
2. 弹出「结束卡片」模态（或独立小窗）：
   - **备注**：多行文本，默认空；占位符示例：「完成了什么」。
   - **关联任务**：下拉或搜索，数据源为飞书「任务表」中未归档/未完成条目（状态字段定义见 5.2）。
   - **任务状态**：若已关联任务，展示可变更状态（如：待办 → 进行中 → 已完成）；未关联则灰置。
3. 用户点击「保存」：
   - 在「番茄记录表」新增一行（见 5.1）。
   - 若关联任务且状态有变更，更新「任务表」对应行。
4. 关闭卡片后回到待机，可一键开始下一番茄。

### 3.3 场景 C：中途放弃

1. 用户确认「放弃当前番茄」。
2. **MVP 策略（建议）**：不写入「已完成番茄表」，或写入「已放弃」类型记录（二选一，需在实现前锁定；推荐 **写入带状态字段** 便于日后分析中断率）。

### 3.4 场景 D：任务列表维护

- **MVP**：应用内任务为 **只读拉取 + 状态更新 + 可选简单筛选**（按状态/标题搜索）；**不在应用内创建复杂子任务树**（创建任务仍在飞书完成）。
- **后续**：应用内快速新建任务、标签、优先级等。

## 4. 功能需求详述

### 4.1 番茄计时（P0）

| ID | 需求 | 说明 |
|----|------|------|
| T-01 | 开始/暂停/继续 | 暂停不计入「完成」；继续后总倒计时不重置（**MVP 可简化为不支持暂停**，以降低与「25 分钟有效专注」的语义冲突；若支持暂停需在 UI 明确标注） |
| T-02 | 放弃 | 二次确认；结果见 3.3 |
| T-03 | 默认时长 | 25 分钟，可配置（写入本地偏好，不必同步飞书） |
| T-04 | 通知 | 使用系统通知 API；声音可关 |
| T-05 | 托盘/菜单栏 | **强烈建议 macOS MVP**：菜单栏图标显示剩余时间或状态（隐藏 Dock 图标可选） |

**建议 MVP 锁定**：首版 **不支持暂停**，仅「开始 / 放弃 / 正常结束」，减少状态机与飞书记录歧义。

### 4.2 结束卡片与任务联动（P0）

| ID | 需求 | 说明 |
|----|------|------|
| E-01 | 备注 | 字符串，建议长度上限 2000 字；落库到番茄记录表 |
| E-02 | 关联任务 | 单选；可选空 |
| E-03 | 更新任务状态 | 仅当有关联任务时可用；枚举与飞书任务表「状态」字段一致 |
| E-04 | 校验 | 保存时若无网络，进入离线队列（6.3）；保存成功给出轻量 Toast |

### 4.3 飞书多维表格集成（P0）

| ID | 需求 | 说明 |
|----|------|------|
| F-01 | 鉴权 | 使用飞书开放平台 **自建应用** + 用户授权（OAuth）或 **个人访问令牌**（仅自用 MVP 可考虑 PAT，安全性与过期策略需在实现文档中写明） |
| F-02 | 配置 | Base（多维表格）ID、各表 ID 或名称映射、字段映射可在首次向导中配置，存本地加密偏好 |
| F-03 | 同步 | 启动时拉取任务列表；可手动「刷新任务」 |
| F-04 | 限流与重试 | 遵守开放平台频率限制；指数退避；用户可见错误文案 |

### 4.4 设置与关于（P1）

- 飞书应用凭证、表格定位信息、字段映射。
- 默认番茄时长、通知开关。
- 关于页：版本号、开源协议（若开源）。

## 5. 数据设计（飞书 Bitable）

以下为多张表分工的 **逻辑模型**；实际字段类型需与飞书字段类型对齐（文本/单选/日期/数字/关联等）。

### 5.1 表：番茄记录（Sessions）

| 字段名（逻辑） | 类型建议 | 必填 | 说明 |
|----------------|----------|------|------|
| 记录 ID | 自动编号 | 是 | 飞书自增或公式 |
| 开始时间 | 日期时间 | 是 | ISO 8601 语义 |
| 结束时间 | 日期时间 | 是 | 正常结束或放弃时写入 |
| 计划时长(分钟) | 数字 | 是 | |
| 结果 | 单选 | 是 | 枚举：`completed` / `aborted` |
| 备注 | 文本 | 否 | |
| 关联任务 | 关联记录 → 任务表 | 否 | 若飞书不支持关联字段，可用「任务行 ID」文本字段代替 |
| 客户端记录 UUID | 文本 | 否 | 用于幂等与离线去重 |

### 5.2 表：任务（Tasks）

| 字段名（逻辑） | 类型建议 | 必填 | 说明 |
|----------------|----------|------|------|
| 任务标题 | 文本 | 是 | 列表展示主键 |
| 状态 | 单选 | 是 | 与 PRD 中枚举一致，如：待办 / 进行中 / 已完成 |
| 优先级 | 单选 | 否 | P1 |
| 截止日期 | 日期 | 否 | P1 |
| 最后由番茄更新 | 日期时间 | 否 | 可选，便于审计 |

**应用侧拉取策略**：默认只拉取状态 ≠「已完成」且未删除的行；支持按标题搜索（客户端过滤或服务端能力以实际 API 为准）。

### 5.3 表：用户偏好（可选，P2）

若希望多端一致配置，可单独一张「配置表」；**MVP 推荐仅本地存储**，减少读写与隐私面。

### 5.4 幂等与冲突

- 每次本地生成 `client_uuid`，写入番茄记录；重试时携带同一 UUID，服务端或应用层去重。
- 任务状态以「最后写入为准」；**MVP 不实现 OT/CRDT**，冲突时提示用户并在日志保留详情。

## 6. 非功能需求

### 6.1 平台与 UI

- **主平台**：macOS 13+（Apple Silicon + Intel 二选一或通用二进制，在技术方案定夺）。
- **UI 调性**：小窗口、低密度信息、圆角与间距一致；支持浅色/深色跟随系统。
- **辅助功能**：键盘可操作主要路径、Tab 顺序合理（P1 可补齐完整 VO）。

### 6.2 体积与分发

| 指标 | 目标（建议） |
|------|----------------|
| 安装包 | MVP 阶段可接受 **约 30–80 MB**（含运行时）；持续优化为后续项 |
| 依赖 | 用户无需安装 JDK；使用 **jpackage / 类似方案** 或所选技术的单文件/安装包 |
| 更新 | **MVP 手动下载新版本**；自动更新为 P2 |

### 6.3 离线

- 本地 SQLite 或同类轻量存储：队列保存「待同步写飞书」的操作。
- 恢复网络后后台刷新；失败条目可展示「重试」入口。

### 6.4 安全

- Token 存系统钥匙串（macOS Keychain）或平台等价物；配置文件不存明文密钥。
- 日志默认脱敏（不打印完整 token）。

### 6.5 计时与系统休眠

- **定义**：番茄进行中若用户合盖或休眠，**MVP 推荐**：计时暂停或冻结（需产品确认）；**不推荐**在休眠期间静默消耗番茄时间。
- 唤醒后提示「休眠已暂停计时」并允许继续或放弃。

## 7. 技术栈建议（回应 BRD 中的 Java 疑问）

### 7.1 Java 25 + 桌面 UI

- **JavaFX**：跨平台成熟，与 Java 版本独立演进（需注意模块与打包）。
- **JetBrains Compose Multiplatform for Desktop**：声明式 UI，Kotlin 为主，可调用 JVM 库；**若团队 Java 远强于 Kotlin，学习成本需计入**。
- **打包**：`jlink` + `jpackage` 可生成 **原生安装包**（`.dmg` / `.msi`），**无需用户安装 JRE**，但安装包仍会包含 **精简过的 JVM**，体积通常 **大于极简 Electron 优化版**，小于「携带完整 JDK」。

**结论**：Java **可以**做小而美的桌面 UI，且可 native 分发；「包不能太大」的极致要求下，Java 不是最小，但在 **可维护性 + 你熟悉 Java** 的前提下是 **合理选项**。

### 7.2 其他主流选项（简要对比）

| 方案 | 包体与分发 | 可维护性 | 备注 |
|------|------------|----------|------|
| **Tauri 2 + Web 前端** | 小，单包友好 | 需 Rust 构建链 + 前端栈 | 体积极优，适合「很简单但要小」 |
| **Flutter Desktop** | 中等 | Dart 单语言 | UI 精致，但 JVM 团队迁移成本高 |
| **Electron** | 偏大 | JS/TS 生态大 | 若不极致控体积可考虑 |

**PRD 级建议**：若以 **Java 技能栈优先**，采用 **Java 21+ LTS + JavaFX 或 Compose Desktop（二选一）+ jpackage**；若将 **包体与启动速度** 列为最高优先级，则评估 **Tauri**。

*具体选型在《技术方案》中结合 CI 与签名公证成本最终确定。*

## 8. MVP 范围边界

**包含**：单用户、单番茄流、结束卡片、飞书双表读写、macOS 安装包、离线队列、菜单栏优先交互（若工期紧可降级为普通窗口）。

**不包含**：团队协作、多 Base 切换、复杂报表、Windows 构建（架构预留即可）、移动端。

自用版可弱化；若需迭代决策，保留本地计数即可：每日完成番茄数、同步失败率、平均备注长度。

## 10. 风险与开放问题

| 风险 | 缓解 |
|------|------|
| 飞书 API 变更 | 字段映射配置化；版本化 SDK 调用封装 |
| OAuth 复杂度 | MVP 可考虑 PAT + 文档说明轮换方式 |
| 计时与休眠语义 | 需在发布前与用户确认一种固定行为并写进用户文档 |

**开放问题（需你确认）**

1. 放弃番茄是否写入记录表？写入时「结果」字段取值是否用 `aborted`？  
2. MVP 是否坚持「暂停」功能？  
3. 飞书侧是否已存在固定 Base，还是允许在向导里新建空表模板？

## 11. 验收清单（MVP）

- [ ] 完成一次 25 分钟番茄并收到系统通知。  
- [ ] 结束卡片保存后，飞书「番茄记录表」新增一行，字段与配置一致。  
- [ ] 关联任务并改状态后，「任务表」对应行状态更新。  
- [ ] 断网保存后联网，记录最终出现在飞书且无重复行（UUID 去重）。  
- [ ] macOS 全新机器无 JDK 条件下可安装并启动。  
- [ ] 密钥不以明文出现在用户可读配置文件中。
