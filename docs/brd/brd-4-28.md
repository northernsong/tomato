# 需求目标

## 背景
传统的番茄钟，要么收费，要么不满足需求。所以决定自己做一个，适配自己的番茄钟app

## 目标
1. 番茄钟的基础功能要有
2. 每个番茄钟结束时，可以填写备注，说明做了什么，也可以快捷的从自己的而任务列表里面关联待完成的任务，以及更改其状态
3. 数据源，使用飞书的多维表格，多维表格中有分不同的数据源，如已完成的番茄钟，任务表格等


## 应用要求
- app的ui要小巧精致，是桌面上的应用，主要用于macOS，windows有空再迭代一个版本，所以需要多平台分发
- 使用语言支持夸平台，且是主流技术
- 我熟悉的技术栈是java，java-25的ui效果如何？如果不行，其他哪些可以。其次要好维护，代码易懂
- 打出的包不能太大，毕竟很简单的东西
- 也易于分发，安装简单。不需要额外的依赖，如果是java的话，应该可以native吧
